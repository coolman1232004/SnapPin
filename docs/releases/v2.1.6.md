# SnapAnchor 2.1.6

SnapAnchor 2.1.6 replaces the remaining mixed-DPI coordinate paths with a single physical-pixel display pipeline.

## Fixes

- Capture, crop, OCR, recording, long scrolling capture, pin placement, and toolbar placement now share native physical monitor bounds.
- Windows is explicitly told that SnapAnchor is Per-Monitor V2 DPI aware, preventing secondary-monitor coordinates from being virtualized.
- Captured bitmaps are normalized to 96-DPI metadata without resizing or resampling their pixels.
- Capture and pin surfaces use device-pixel alignment and nearest-neighbor rendering for native-size sharpness.
- The pinned-image toolbar stays effectively hidden while Windows completes the first destination-monitor DPI transition, then appears beside the pin in its final position.
- Existing native-bound stabilization is retained so a newly pinned selection stays at the captured location and size.

The monitor approach was independently adapted from the public physical-coordinate design used by Snow Shot; no Snow Shot source code or assets are included.

## Downloads

- **Installer:** `SnapAnchor-Setup-win-x64.exe`
- **Portable:** `SnapAnchor-Portable-win-x64.zip`

The installer and portable build are currently unsigned, so Windows SmartScreen may display a warning. SnapAnchor requires 64-bit Windows 10 version 1809 or later, or Windows 11.
