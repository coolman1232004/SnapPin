## SnapPin 1.2.7

SnapPin 1.2.7 gives portable updates a complete, predictable workflow.

### What changed

- Startup checks are quiet. An available update appears as a Windows notification and a highlighted tray command.
- The review screen shows the new version, package size, release notes, and a GitHub link before downloading.
- Download progress includes transferred size, total size, speed, percentage, and estimated time remaining.
- Downloaded packages are SHA-256 verified and portable files are extracted into a safe staging folder.
- After preparation, choose **Restart and update** or **Later**. Choosing Later keeps the verified update ready without downloading it again.
- A visible updater waits for SnapPin to close, backs up managed files, applies the update, verifies the executable version, and reopens SnapPin.
- SnapPin confirms a successful update after restart. A failed update restores the prior version and reports the local update log.

### Portable safety

- File replacement is transactional: overwritten and obsolete managed files are backed up, while newly created managed files are tracked for rollback.
- A package manifest removes obsolete SnapPin files without touching user-created files in the portable folder.
- Read-only locations request administrator permission, free space is checked before replacement, and only the newest two rollback copies are retained.

### Validation

- Real filesystem tests verify both successful replacement and forced-failure rollback.
- 526 localization keys and 467 visible UI strings audited across English, Simplified Chinese, and Traditional Chinese.
- Full x64 release build and smoke-test suite pass with zero warnings or errors.
