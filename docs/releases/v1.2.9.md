## SnapAnchor 1.2.9

SnapAnchor 1.2.9 fixes the access-denied failure discovered while applying the 1.2.8 portable update.

### Fixed

- If Windows denies replacement of an existing portable application file, SnapAnchor first restores the previous version and then asks for administrator approval to retry safely.
- The retry is not offered if rollback did not complete or if the updater is already elevated.
- Failure messages and the update log now identify the exact backup, replacement, or removal operation that failed.

### Testing from portable SnapAnchor 1.2.8

Open **Preferences > About > Check now**. Download 1.2.9 and choose **Restart and update**. If the portable folder requires additional permission, approve the Windows administrator prompt; SnapAnchor should then finish the update and reopen automatically.
